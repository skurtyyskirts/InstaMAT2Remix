# InstaMAT2Remix (RTX Remix Connector for InstaMAT)

This plugin bridges **InstaMAT** and **NVIDIA RTX Remix**, allowing you to **pull meshes/textures** from Remix and **push textures** generated by an InstaMAT graph back into Remix materials.

## Features

- **Pull from Remix**: **Automatically creates a new InstaMAT texturing project** using the selected mesh from RTX Remix. The project is linked for future push/pull operations. Optionally imports textures automatically after project creation.
- **Import Textures from Remix**: Pulls linked (or selected) material textures from RTX Remix. Converts `.dds`/`.rtex.dds` to `.png` via `texconv.exe`.
- **Push to Remix**: Executes a selected InstaMAT graph, exports matching outputs, ingests textures in Remix, and updates the target material.
- **Force Push to Remix**: Push to a *different* currently selected Remix material (relinks).
- **Auto-unwrap with Blender (optional)**: Smart UV unwrap on pull before project creation.
- **Settings UI**: Tabbed settings dialog with browse buttons and a one-click connection test.
- **Diagnostics**: A diagnostics report dialog (copy-to-clipboard) to troubleshoot paths/connection/settings.
- **Logging**: Persistent log file written to `Documents/InstaMAT2Remix/logs/remix_connector.log`.

## Requirements

- **InstaMAT Studio** (with Plugin SDK support)
- **RTX Remix Toolkit** running with its REST service (default: `http://localhost:8011`)
- **Blender** (optional, for auto-unwrap)
- **texconv.exe** (included next to the plugin DLL)

## Installation (developer)

Run the build + install script:

- `InstaMAT2Remix/build_plugin.ps1`

It builds `InstaMAT2RemixPlugin.dll` and installs it into common InstaMAT plugin locations (including a dev junction under `Documents/InstaMAT/Plugins/InstaMAT2Remix`).

## Usage

In InstaMAT, use the **RTX Remix** menu.  
If InstaMAT already has an RTX Remix menu (e.g. from a built-in connector), this plugin will appear as a submenu: **RTX Remix â†’ RTX Remix Connector**.

### Pull from Remix

1. In RTX Remix, select a mesh/material.
2. In InstaMAT: **RTX Remix â†’ Pull from Remix**
3. The plugin will:
   - Automatically create a new "Asset Texturing" project with the selected mesh
   - Link the project to the selected Remix material
   - Optionally auto-import textures (if enabled in Settings)
   - Copy the mesh path to clipboard as a fallback

### Import Textures from Remix

1. In InstaMAT: **RTX Remix â†’ Import Textures from Remix**
2. Textures are saved to a pulled-textures folder and registered as an external asset folder in InstaMAT.

### Push to Remix

1. In InstaMAT: **RTX Remix â†’ Push to Remix**
2. Select the graph to export (first time only; persisted).
3. The plugin executes the graph and exports output images by matching common output names:
   - `Albedo`, `Normal`, `Roughness`, `Metallic`, `Emissive`, `Height`, `Opacity` (optional)
4. It ingests the textures into the Remix project output directory and updates the material bindings.

### Force Push to Remix

1. In RTX Remix, select a different target material.
2. In InstaMAT: **RTX Remix â†’ Force Push to Remix**

## Tips

- **Output naming matters**: If no textures are exported, rename your graph outputs to match the expected names above.
- **Opacity**: Controlled by the **Include Opacity Map** setting (disabled by default).
- **Timeouts**: Increase **Request Timeout** in Settings if your Remix project is large or slow to respond.
